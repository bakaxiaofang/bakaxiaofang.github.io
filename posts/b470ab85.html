在Windows下使用win-acme自动申请免费的Let's Encrypt泛域名证书

Created2024-02-01|Updated2024-02-03|建站证书
|Post Views:

一、下载win-acme

1.客户端选择

Let’s Encrypt 官网
本来官网是推荐用Certbot客户端的,但是这个的Windows版本没法安装插件certbot-dns-cloudflare
ACME客户端选择详情请参阅ACME客户端
DNS插件参阅certbot DNS插件
详情参阅GitHub上提交的issues
issues9589
issues8048
接着还想着使用acme.sh,但是需要安装Cygwin,我嫌麻烦就算了’W’
win-acme 官网
win-acme GitHub
certbot 官网
certbot GitHub
acme.sh GitHub

2.下载win-acme和dns cloudflare插件

在win-acme 官网或GitHub 页面下载
!注: 选择pluggable版本,trimmed 较小版本不支持插件.
我在GitHub页面下载的版本 win-acme.v2.2.7.1612.x64.pluggable.zip
将下载的压缩包随便放在一个文件夹内并解压.
插件下载页面
插件问题具体请参阅win-acme Cloudflare
插件下载地址
2.2.7cloudflare直连下载
将下载的插件解压到pluggable版本的win-acme根目录里.
文件如下
win-acme.v2.2.7.1612.x64.pluggable
-plugin.validation.dns.cloudflare.v2.2.7.1612
-FluentCloudflare.dll
-Newtonsoft.Json.dll
-PKISharp.WACS.Plugins.ValidationPlugins.Cloudflare.dll
-Scripts
-各种脚本
-public_suffix_list.dat
-settings_default.json
-version.txt
-wacs.exe
-Web_Config.xml

二、创建API令牌并自动申请证书

1.创建API令牌

创建Cloudflare API令牌

2.申请Let’s Encrypt泛域名证书

右键win-acme.exe以管理员打开
输出如下(以下中文为谷歌机翻)
这里要注意一下,如果用的不是IIS,就不要选择N：创建证书（默认设置）
选择’m’ 创建证书（全选项）

选择 '2' 手动输入域名

Host: 输入需要证书的泛域名
例如: '*.bakaxiaofang.moe'
然后回车

选择'1' 每个域都有单独的证书（例如*.example.com）

选择 '6' [dns]在Cloudflare DNS中创建验证记录

选择"1" 从控制台中输入/粘贴令牌

Cloudflare API Token:
复制并粘贴自己的令牌
最后输入y保存到保管库以供将来重复使用
名称随意

我思考了一下,选择'1' ECC密钥,感觉比RSA好.

我这里没有以管理员打开win-acme,所以没有IIS
我这里用的是nginx反代
所以选择'2' PEM编码的文件（Apache、nginx等）
文件路径: 绝对路径
例如:E:\Server\Website\SSL\bakaxiaofang.moe

为私钥 .pem 文件设置的密码。
选择'1'
然后选择'5'
再选择'3'
服务条款想看就y打开

如果出现 Failed to create task
那是因为你输入的不是微软用户名和密码
密码不是PIN,是你微软账户的密码
用户名C:\Users可以看见
正常情况下,如果已经登录微软账户
用户名为前五个字母 
在O: More options... "更多选项"
 T: (Re)create scheduled task ""（重新）创建计划任务"
